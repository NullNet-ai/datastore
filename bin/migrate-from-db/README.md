# migrate-from-db

Migrates data from a PostgreSQL database (**migrate_from**) into the store API (**migrate_to_store_url**) using the store’s create route.

1. **Login once** – POST to `/api/organizations/auth?app_id=portal` with credentials; receives a token.
2. **Read from Postgres** – For each table, runs `SELECT row_to_json(t) FROM "table" t` (raw SQL, no ORM).
3. **Insert into store** – For each row, POST to `/api/store/{table}?pluck=id,code` with `Authorization: Bearer <token>` and the row as JSON body. The same token is reused so the session is not re-created on every request.

Table order is read from the file generated by **order-tables** (`migrate_tables_order.rs`). Run `make order-tables` first, then run the migration.

**Source DB is read-only:** The migration only runs `SELECT` against `MIGRATE_FROM_DATABASE_URL`. The connection is set to `default_transaction_read_only = on`, so no INSERT/UPDATE/DELETE can run on the source (safe for QA). Writes go only to the store API (`MIGRATE_TO_STORE_URL`).

## Env (required)

| Variable | Description |
|----------|-------------|
| `MIGRATE_FROM_DATABASE_URL` | Postgres connection string (source DB). |
| `MIGRATE_TO_STORE_URL` | Store base URL (e.g. `http://localhost:3002`). |
| `MIGRATE_STORE_ACCOUNT_ID` | Store login `account_id` (e.g. `admin@dnamicro.com`). |
| `MIGRATE_STORE_ACCOUNT_SECRET` | Store login `account_secret`. |

## Optional

| Variable | Description |
|----------|-------------|
| `MIGRATE_APP_ID` | Auth query param; default `portal`. |
| `MIGRATE_TABLES_FILE` | Path to the table-order file; default `migrate_tables_order.rs` (from order-tables). |
| `MIGRATE_ERROR_LOG` | File to append every error; default `migrate_errors.log`. Migration continues on row errors and exits non-zero if any were logged. |

## Example

```bash
cd bin/migrate-from-db
make order-tables   # generate migrate_tables_order.rs from DB FK relations
make run            # run migration (reads tables from that file)
```

Or run the binary directly; ensure `migrate_tables_order.rs` exists in the current directory (or set `MIGRATE_TABLES_FILE`).

## order-tables (generate table order from DB)

A second binary in this crate connects to the database, reads **all tables and foreign-key relations** from `information_schema`, and **topologically sorts** them so that referenced tables come first. It writes the result as a **Rust array** to a file. The migration binary reads the table list from this file (no `MIGRATE_TABLES` env needed).

### Run via Makefile (from this folder)

```bash
cd bin/migrate-from-db
make order-tables
```

Uses `MIGRATE_FROM_DATABASE_URL` from `.env` or env. Writes `migrate_tables_order.rs` in this folder by default. Override with `MIGRATE_ORDER_OUTPUT=path/to/out.rs make order-tables`.

### Run directly

```bash
cd bin/migrate-from-db
export MIGRATE_FROM_DATABASE_URL="postgres://user:pass@localhost:5432/yourdb"
cargo run --bin order-tables
```

### Env

| Variable | Description |
|----------|-------------|
| `MIGRATE_FROM_DATABASE_URL` or `MIGRATE_ORDER_DATABASE_URL` | Postgres URL (required). |
| `MIGRATE_ORDER_SCHEMA` | Schema to scan; default `public`. |
| `MIGRATE_ORDER_OUTPUT` | Output file path; default `migrate_tables_order.rs`. |

### Output

The script writes a Rust array, e.g.:

```rust
// Generated by migrate-from-db order-tables - do not edit by hand
// Tables in dependency order (referenced tables first). Use for MIGRATE_TABLES.

pub const MIGRATE_TABLES_ORDER: &[&str] = &[
    "organizations",
    "accounts",
    "samples",
    // ...
];
```

Order is strict: for every foreign key (A → B), table B appears before table A. If a cycle is detected, the script errors.
